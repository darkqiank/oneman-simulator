## 🎮 游戏机制

### 📈 收入与成本
- **收入来源**：用户订阅费用（按月计算，游戏中按天结算）
- **成本支出**：服务器日常维护费用
- **盈利公式**：`净利润 = 订阅收入 - 维护成本`

### 👥 用户增长
影响因素：
- 套餐价格与配置的性价比
- 口碑值（Reputation）
- 营销活动加成
- 超售率（过高会降低增长）

### 😭 用户流失
**订阅周期机制**：
- 用户订阅周期为 30 天
- 每天约有 1/30 的用户到期需要续费
- 续费率取决于口碑值和服务质量

**续费率影响因素**：
- **口碑值**：60 分 → 70% 续费，100 分 → 90% 续费
- **超售率**：超售越高，续费率越低
- **DDOS 攻击**：严重影响续费意愿
- **新手保护期**：前 30 天没有流失

### ⚖️ 超售策略
这是游戏的核心机制：
- **低超售 (< 1.5x)**：安全但利润较低
- **中超售 (1.5x - 2.5x)**：平衡风险与收益
- **高超售 (> 2.5x)**：高利润但用户流失严重

### 🎨 主题系统
- **Cyber 主题** - 赛博朋克风格（默认）
- **Scam 主题** - 复古 Geocities 风格（解锁）

## 🚀 快速开始

### 环境要求
- Node.js 16.0+
- npm 或 yarn

### 本地运行

1. **克隆项目**
```bash
git clone <repository-url>
cd oneman-simulator
```

2. **安装依赖**
```bash
npm install
```

3. **启动开发服务器**
```bash
npm run dev
```

4. **打开浏览器**
访问 `http://localhost:5173` 开始游戏！

### 构建生产版本
```bash
npm run build
```

生成的文件将位于 `dist/` 目录。

## 🎯 游戏提示

### 新手指南
1. **第一步**：游戏开始时会赠送一台 Starter 服务器和基础套餐
2. **稳定发展**：前 30 天有新手保护期，不会有用户流失
3. **逐步扩张**：先稳定运营，有盈利后再采购新服务器
4. **关注口碑**：口碑值影响用户增长，及时处理工单
5. **谨慎超售**：超售率不要超过 2.0x，否则流失会很严重

### 进阶策略
- 🌍 **区域布局**：不同地区的服务器有不同的成本和定价空间（香港最贵但价格也高）
- 💡 **套餐组合**：提供不同价位的套餐，覆盖更多用户群体
- 📊 **数据分析**：密切关注超售率和用户流失，及时调整策略
- 🔧 **优先科研**：尽早解锁自动工单和防护科技，节省时间
- 🎯 **营销时机**：现金充裕时投放广告，快速扩大规模

### 高手技巧
- 在高口碑时适当提高超售率，利润最大化
- 利用 DDOS 攻击的间隙期大规模营销
- 工单优先解决 VIP 用户的问题（口碑奖励更高）
- 合理使用"封禁"功能处理恶意用户

## 🎨 自定义开发

### 添加新服务器
编辑 `constants.ts` 中的 `HARDWARE_CATALOG`：
```typescript
{
  id: 'your-server',
  name: 'Your Server Name',
  region: 'US',
  cpuCores: 8,
  ramGB: 16,
  diskGB: 500,
  // ...
}
```

### 添加新成就
在 `constants.ts` 的 `MILESTONES` 中添加：
```typescript
{
  id: 'achievement-id',
  name: '成就名称',
  description: '成就描述',
  condition: (state) => state.cash >= 100000,
  rewardCash: 5000,
  achieved: false
}
```

### 修改游戏平衡
调整 `gameLogic.ts` 中的各项参数：
- 用户增长速率
- 流失率计算
- 收入计算方式
- 工单生成频率
